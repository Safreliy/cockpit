# Web Cockpit Prototype

This folder contains the cleaner cockpit interface. It is separate from the generated MVP artifact in `output/cockpit_mock.html`.

Run the simulator first:

```bash
python run.py
```

Then open:

```text
web_cockpit/index.html
```

The app reads `web_cockpit/data.js`, which is regenerated by `run.py`.

## UX Model

- The first screen stays compact: live stream status, one chart, current detections, and current key signals.
- Telemetry is replayed as an online stream instead of shown only as a post-run report.
- The UI transitions from healthy to anomaly detected to causal episode detected.
- The runtime demo spans two hours and contains multiple incidents.
- Detected incidents are clickable markers on the chart.
- The Detections panel is a scrollable list of incidents detected by the current stream time; future incidents are not shown.
- Selecting a marker highlights its investigation window and opens the matching causal report.
- Selecting a detection list item does the same thing as selecting its chart marker.
- Detailed diagnosis, evidence, and rejected causes live in the report drawer.
- Later this static playback can be replaced by WebSocket or SSE without changing the top-level interaction model.
